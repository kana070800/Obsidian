const 용법
	기본 자료형의 앞, 뒤에 const 지정 시 기본 자료형의 변경 불가
		`char const *ap 의 경우 char를 const >> 주소는 변경 가능
		`const char *ap 의 경우 char를 const >> 주소는 변경 가능
	포인터 뒤에 const 지정 시 포인터(주소값) 변경 불가
		`int *const a` 의 경우 주소(a) 변경 불가
	주소와 자료형 둘 다 const 지정 가능
		`const int* const a` 의 경우 둘 닷 변경 불가
	`#define 와 같은 메크로의 경우 지정 위치에 따라 전처리 결과가 달라질 가능성이 있다
		const가 컴파일러 검사 단계를 거치기에 좀 더 안정적
	const char* pc = "mango"; 과 같이 const 를 지정하여 오류 검출 가능 `(pc[0] = 't')
