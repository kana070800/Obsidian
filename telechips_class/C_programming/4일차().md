	sizeof, & 연산일 때 배열 전체 공간을 가리키는 배열포인터로 인식
	
	`int arr[3][4]`
		`arr == &arr[0]` >> 1차원 배열 포인터
			arr 요소의 수 == 3개, 요소 type == int [4]
		`&arr` 2차원 배열 포인터


차원이란 하나의 정보에 접근하기 위한 정보의 가짓수로 이해 
	`int a[][][] 와 같은 배열 형태의 경우 다른 방식으로 구조화 해보자`
	메모리에 저장된 정보에 어떻게 엑세스할까, 접근 양상에 따라 다르게 재정의하여 참조가능
	포인터는 언제든 형변환이 가능하다(배열의 차원까지도)
		2차원 배열을 1차원 배열로 형변환하여 참조

`int (*p[4])[5]`
	요소의 수 = 4
	요소의 타입 = `int (*)[5]`
	'배열 포인터'의 배열
`int *p[4]`
	요소의 수 = 4
	요소의 타입 = int *
	int 포인터의 배열
`int (*p[3])(int, int)`
	요소의 수 = 3
	요소의 타입 = `int (*)(int, int)`
	함수포인터의 배열
`int (*(*ap)())()`
	함수 포인터를 반환하는 함수 포인터
`char *(*(*var)())[10]` 
	포인터의 배열 포인터를 반환하는 함수 포인터
