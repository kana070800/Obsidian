배열등가 포인터의 생성
	식별자 뒤 요소의 수 제거
	식별자앞에 *추가하고 괄호로 묶기

배열을 함수 인자로 넣을 때 포인터와 동치라고 들었다
포인터를 함수 인자로 넣으면 call by reference 이기에 값의 변경이 반영된다고 아는데
배열을 함수 인자로 넣어도 call by reference로 적용되어서 변경이 반영되는가?
	C언어에서 배열은 통째로 복사해서 넘길 수 없습니다. 만약 배열의 내용을 보호하고 싶다면(원본 변경 금지), 매개변수에 const 키워드를 붙여야 합니다.

`int  (*p[5])[3]
와 같이 포인터의 배열의 크기는 4*5 이다
`함수인자로 넘길 시 int (**p)[3] 의 형식으로 이중포인터를 사용하여 넘겨야 한다

`int  (*p[5][3])(int) >> int  (*(*p)[3])(int) 
함수 포인터의 배열의 포인터

이중포인터의 경우 포인터 변수에 저장된 주소값을 바꾸고 싶을 때 참조를 통해 바꾸기 위해 함수의 인자로 사용할 수 있다.

가변 크기 배열의 크기 정보를 넣고 활용해야 할 때, 일반적으로 0번 자리에 배열의 크기정보를 넣고 활용하는 편이다
