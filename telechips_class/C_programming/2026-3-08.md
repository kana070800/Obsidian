배열 등가 포인터
	1차원 배열의 경우 단순 포인터
	2차원 배열을 함수의 인자 `int a[2][3]` >>int 3개 배열을 가리키는 포인터로 인지
	배열 등가 포인터는 괄호로 배열포인터로서 인지하도록 조정 `int a[][] > int (*a)[]`
		괄호가 없다면 포인터의 배열로 인지할 수 있다(이중포인터) > `int *a[]`
		`int a[2][3][4] = {0}; > int (*pa)[3][4] = a;` 이차원 배열의 포인터

`int a[3][4]`
	`a == &a[0] >> int (*)[4]`
	`a[0] == &a[0][0] >> int *`
	`a[1] == &a[1][0] == a + 1 >> int *`
	`a[2] == &a[2][0] == a + 2 >> int *`  
	하나의 요소를 이동원할 시 >> `a[1] + 1`
	`&a == 48 , &a + 1 은 48byte 이후의 주소`
	예시
```
	printf("%d %d\n", sizeof(*a), sizeof(*a[0])); //첫번째 요소값의 크기
	printf("%d %d %d", sizeof(a), sizeof(a[0]), sizeof(a[0][0])); //배열의 크기
	printf("%d %d", sizeof(&a), sizeof(&a[0])); // &은 포인터이므로 4
	
	printf("%p ", &a); //배열전체
	printf("%p ", &a+1);//다음 배열크기 건너뜀
	printf("%p ", a);   //첫번째 요소 주소
	printf("%p ", a+1); //두번째 요소 주소
	printf("%p ", a[0]);
	printf("%p ", a[0]+1); //바로 위 코드와 비용은 동일(상수연산)
	printf("%p ", a[1]);
	printf("%p ", a[1]+1);
	//00BAFC44 00BAFC74 00BAFC44 00BAFC54 00BAFC44 00BAFC48 00BAFC54 00BAFC58
```
포인터의 주소값은 동일해도 주소가 가리키는 크기가 달라질 수 있으니 해당 사실 고려가 중요
	어려우면 싹 괄호로 묶어버리고 형변환하는게 편해보인다

`배열의 이름이란 첫번째 요소를 가리키는 포인터`

포인터 형식으로 통일
배열이름은 첫번째 요소를 가리킴, 각 요소의 byte 수 접근
	배열 이름은 n-1차원을 가리킴, -1차원일 때 실제 인수, 0차원은 단순 포인터
배열이름 + 상수 연산을 통해 각 배열 요소를 접근, `*`에 따라 차원 감소, &으로 차원 증가

`char *ap[n]` 은 포인터의 배열이므로 sizeof(ap)는 `4*n` 으로 계산된다.
	192p의 malloc 을 통해 임의의 크기의 heap영역을 할당 받을 수 있다.
